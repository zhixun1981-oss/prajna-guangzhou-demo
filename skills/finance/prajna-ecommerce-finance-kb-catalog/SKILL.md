# prajna-ecommerce-finance-kb-catalog

## 技能定位

本技能是 Prajna 企智 `finance` 方向原生能力，面向电商企业，一键生成标准化的 **财务知识库钉钉文档目录 Markdown 文件**。

目录采用 `数字_中文` 命名规范，包含七大一级模块：

1. 核算规范
2. 成本利润模型
3. 库存管理
4. 资金税务
5. 经营分析
6. 内控审计
7. 工具模板

每个模块下设 3 个子文件夹，首页包含醒目标题、版本信息、适用对象、维护部门与更新日志说明。

适用于：电商企业财务负责人、财务共享中心、运营财务 BP、内控审计人员以及需要在钉钉文档中快速搭建财务知识库的团队。

## 触发条件

- 用户说「生成电商财务知识库目录」「财务知识库钉钉文档模板」「电商财务知识库框架」等
- 关键词命中：电商、财务、知识库、钉钉文档、目录、核算、成本、库存、税务、经营分析、内控、审计、模板
- CLI 调用：`prajna skill run prajna-ecommerce-finance-kb-catalog`
- 其他 skill 通过协同调用规范触发（见下方）

## 执行流程

1. **确认企业信息**：企业名称、维护部门/人、更新日期。
2. **生成 Markdown 目录**：
   - 首页标题区：企业名称 + 知识库标题、版本、适用对象、维护部门、最近更新。
   - 更新日志：版本号、更新日期、更新内容、维护人。
   - 目录结构树：清晰展示 7 大模块与 21 个子文件夹层级。
   - 模块详情：每个模块的定位说明、3 个子文件夹的职能、推荐文档类型。
   - 使用说明与免责声明。
3. **保存并返回文件路径**。

## CLI 参数

通过 `scripts/generate_catalog.py` 调用：

| 参数 | 简写 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| `--company` | `-c` | string | 否 | 企业名称，用于标题，默认 `XX电商` |
| `--author` | `-a` | string | 否 | 维护部门/人，默认 `财务部` |
| `--date` | `-d` | string | 否 | 更新日期，默认当天 |
| `--output` | `-o` | string | 否 | 输出文件完整路径，覆盖默认输出目录 |

## 使用示例

```bash
# 使用默认参数生成
python3 scripts/generate_catalog.py

# 指定企业与维护部门
python3 scripts/generate_catalog.py --company "智云电商" --author "财务共享中心"

# 指定输出路径
python3 scripts/generate_catalog.py \
  --company "锐行电商" \
  --author "财务部·张经理" \
  --output ~/Desktop/电商企业财务知识库目录.md
```

## 输出约定

- **默认输出目录**：`~/.prajna/prajna-ecommerce-finance-kb-catalog/samples/`
- **默认文件命名**：`prajna_电商企业财务知识库目录_<企业名>_<日期>.md`
- **输出格式**：Markdown（`.md`）
- **模板特点**：
  - 严格采用 `数字_中文` 命名，便于钉钉文档按字典序自动排序。
  - 首页醒目标题 + 更新日志，便于知识库使用者快速了解版本与维护信息。
  - 每个模块与子文件夹均附定位说明与推荐文档类型，便于团队按图索骥填充内容。
  - 附带免责声明，提示政策与制度需人工复核。

## 与现有 Skill 的打通说明

### 协同调用规范

本 skill 设计为可被其他 Prajna 原生 skill 调用，统一入口为：

```bash
python3 /Users/a12345/.prajna/skills/finance/prajna-ecommerce-finance-kb-catalog/scripts/generate_catalog.py \
  --company <企业名> \
  --author <维护部门/人> \
  --date <更新日期> \
  --output <输出路径>
```

建议的调用方与场景：

- **`prajna-finance`**：在为企业搭建财务体系时，调用本脚本生成知识库目录作为落地载体。
- **`prajna-report-agent`**：在完成财务分析后，将报告归档到对应模块目录，并更新本目录。
- **`prajna-knowledge`**：在企业知识库初始化或升级时，调用本脚本输出标准目录框架。
- **`prajna-internal-control`**：在内控审计体系建设阶段，调用本脚本生成内控审计模块目录。

调用方应负责提供 `--company`、`--author`、`--date` 等参数；本 skill 仅负责目录 Markdown 生成与文件输出。
